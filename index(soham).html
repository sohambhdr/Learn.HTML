Introduction to HTML

HTML stands for HyperText Markup Language. It's the standard language for creating web pages. HTML describes the structure of a web page using tags and elements.

  • Tags: Keywords surrounded by angle brackets, like <p> (opening) and </p> (closing).
  • Elements: Tags plus their content, e.g., <p>This is a paragraph.</p>.
  • HTML is not case-sensitive, but lowercase is conventional.
  • Web browsers render HTML to display pages.

HTML5 is the latest version, which we'll focus on.

Basic HTML Structure

Every HTML document has a basic skeleton:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Page Title</title>
</head>
<body>
    <!-- Your content here -->
</body>
</html>
  • <!DOCTYPE html>: Declares the document as HTML5.
  • <html>: Root element.
  • <head>: Contains meta information.
  • <body>: Visible content.
  • Comments: <!-- Comment --> (not displayed).

Example:

Hello, World!

Links and Images

Links

Use <a> for hyperlinks.

Visit Example 
<a href="https://example.com" target="_blank">Visit Example</a>
  • href: URL.
  • target="_blank": Opens in new tab.
  • Email link: <a href="mailto:email@example.com">Email</a>
  • Anchor link: <a href="#section">Jump</a> to <div id="section">

Images

Use <img> (self-closing).

<img src="image.jpg" alt="Description" width="150" height="150">
  • src: Image URL or path.
  • alt: Alternative text for accessibility.
  • width and height: Dimensions.

Tables

Use <table> for tabular data.

Header 1 Header 2
Data 1 Data 2 
<table border="1">
    <thead>
        <tr>
            <th>Header 1</th>
            <th>Header 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Data 1</td>
            <td>Data 2</td>
        </tr>
    </tbody>
</table>
  • <tr>: Table row.
  • <th>: Header cell.
  • <td>: Data cell.
  • Attributes like colspan, rowspan for merging cells.

Forms

Forms collect user input with <form>.


<form action="/submit" method="post">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name"><br>
    <input type="submit" value="Submit">
</form>

Input Types

  • type="text": Text field.
  • type="password": Hidden text.
  • type="email": Email validation.
  • type="number": Numbers.
  • type="checkbox": Checkboxes.
  • type="radio": Radio buttons.
  • type="file": File upload.
  • type="submit": Submit button.

Other Form Elements

  • <textarea>: Multi-line text.
  • <select>: Dropdown with <option>.
  • <label>: Labels for inputs.
  • Attributes: required, placeholder, value.

Note: Forms typically send data to a server (action URL), but for learning, we use # as placeholder.

Semantic HTML

Semantic elements provide meaning to content for better SEO and accessibility.

  • <header>: Page header.
  • <nav>: Navigation.
  • <main>: Main content.
  • <article>: Independent content.
  • <section>: Thematic grouping.
  • <aside>: Sidebar.
  • <footer>: Footer.
  • <figure> and <figcaption>: For images with captions.

Attributes, Classes, and IDs

Attributes provide additional info to elements.

  • Global attributes: class, id, style, title.
  • id: Unique identifier (one per page).
  • class: For grouping (multiple elements).
<div id="unique" class="group">Content</div>

Used with CSS/JS: style="color: red;" for inline styles.

Layout and Structure

Use <div> for generic containers (block-level).

Use <span> for inline containers.

Block vs Inline

  • Block: Starts new line (e.g., <div>, <p>).
  • Inline: Doesn't (e.g., <span>, <a>).

For layouts, combine with CSS (but HTML provides base).

Head and Meta Tags

The <head> contains non-visible info.

  • <title>: Browser title.
  • <meta charset="UTF-8">: Character encoding.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Responsive design.
  • <meta name="description" content="Page desc">: SEO description.
  • <link rel="stylesheet" href="styles.css">: Link CSS.
  • <script src="script.js"></script>: Link JS.

Accessibility (a11y)

Make web pages usable for everyone, including disabled users.

  • Use alt on images.
  • Semantic tags for screen readers.
  • aria-* attributes: e.g., aria-label="Description".
  • Proper heading hierarchy.
  • Keyboard navigation: Use tabindex.
  • Color contrast and font sizes.

Tools: Validate with WAVE or Lighthouse.

Best Practices

  • Validate HTML with W3C validator.
  • Use semantic tags over divs.
  • Keep code indented and readable.
  • Separate structure (HTML), style (CSS), behavior (JS).
  • Optimize images and use lazy loading (loading="lazy" on img).
  • Test on multiple browsers/devices.
  • Follow SEO: Use titles, metas, alt texts.
  • Security: Sanitize inputs, use HTTPS.
